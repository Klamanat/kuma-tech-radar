## 🤖 AI / LLM / Agents

**EightX Labs agnt8x — marketplace public สำหรับ recruit, operate, monetize AI agents** [official]

EightX Labs ปล่อย agnt8x เมื่อ 8 มิ.ย. 2026 เป็น public platform ให้ builders สร้าง, test, monetize AI agents ภายใน marketplace เชื่อมต่อกัน ฟีเจอร์หลัก ได้แก่ agent builder (no-code/code), Agent Manifest (EAM) v0.1 specification สำหรับ declare capabilities, multi-agent orchestration conductor, reputation/review system คล้าย GitHub Stars Platform นี้เป็นความพยายามสร้าง "hub" สำหรับ agent developers ลดความยุ่งของการสร้าง agents จาก scratch ตรงกับ trend agentic AI ขึ้นเป็น mainstream architecture ([agnt8x](https://www.agnt8x.io/) · [EightX Labs Blog](https://eightxlabs.ai/agnt8x-public-launch/))

**MetaMask Agent Wallet — AI agents ทำ onchain transactions ด้วย security checkpoints** [official]

MetaMask ปล่อย Agent Wallet เมื่อ 8 มิ.ย. 2026 ให้ AI agents execute autonomous onchain transactions บน EVM chains (Ethereum, Arbitrum, Polygon) ด้วย mandatory security controls: spending limits, transaction templates, human approval checkpoints ได้ execute swaps, stake, bridge assets โดยไม่ต้อง user sign แต่ละ tx ลด friction จาก 15 clicks เหลือ ~2 Early access เปิดให้ beta testers บน Ethereum mainnet ชี้ว่า MetaMask เห็น agentic finance เป็น next frontier ของ DeFi ([MetaMask Blog](https://blog.metamask.io/introducing-metamask-agent-wallet/) · [DefiLlama](https://defillama.com/research/agent-wallet))

**US Government ห้าม Anthropic Fable 5 และ Mythos 5 global access — jailbreak vulnerability** ⚠️

US government order ออกมาช่วงต้นมิ.ย. 2026 สั่ง Anthropic ปิด global access ต่อ Fable 5 และ Mythos 5 inferencing servers หลังกล่าวอ้างว่ามี prompt injection vulnerability ที่ bypass safety guardrails Anthropic response ยืนยันว่า patching แต่ปล่อยให้ operators กำหนด rate-limits และ sandboxing ด้วยตนเอง เซิร์ฟเวอร์ส่วนใหญ่อยู่บน AWS, GCP, Azure globally ประเด็นนี้เตือนให้ audit agent security assumptions ต่างๆ ([SecurityWeekly](https://securityweekly.com/news/fable5-mythos5-jailbreak) · [Anthropic Statement](https://www.anthropic.com/news/safety-updates))

→ **เกี่ยวกับเรา:** agnt8x marketplace ช่วย discovery และ integration ของ agents ทำง่ายขึ้น; MetaMask Agent Wallet เปิด onchain autonomy window ใหม่ สำหรับ DeFi agents; Fable 5 jailbreak เตือนให้ audit agent security

## ☸️ Cloud / DevOps / Infra

**Microsoft Kubernetes Dashboard AI-assisted querying — natural language cluster queries** ⚠️

Microsoft เพิ่มความสามารถเข้า Kubernetes Dashboard ช่วง Q2 2026 ให้ operators query cluster state ด้วย natural language เช่น "show me pods using over 2GB memory" AI summarize ผลลัพธ์ รองรับหลายภาษา (Thai, Japanese, English) ลดเวลา time-to-first-incident-resolution ~30% โดยตัดทอน irrelevant logs ([CloudNativeNow](https://cloudnativenow.com/features/microsoft-simplifies-kubernetes-management-with-ai-integration-2/) · [Kubernetes Dashboard](https://kubernetes.io/blog/2026/06/dashboard-query/))

**AI Workloads on Kubernetes dominant — MLOps platforms, FinOps pressure** ⚠️

CNCF 2026 survey พบ 73% องค์กรที่ใช้ K8s ประกาศว่า heaviest workloads เป็น MLOps (machine learning operations) ต้องการ orchestration ของ resource-intensive Jobs + continuous Services งบประมาณ AI workloads ลดการ scale general compute 15–25% องค์กรที่ adopt FinOps (cost visibility, chargeback) เพิ่มขึ้น 40% YoY GPU/TPU allocation ผ่านเลนส์ cost-per-model ไม่ใช่ cost-per-instance กลายเป็นจำเป็น ([Fairwinds Blog](https://www.fairwinds.com/blog/2026-kubernetes-playbook-ai-self-healing-clusters-growth) · [CNCF Report](https://www.cncf.io/blog/2026-kubernetes-report/))

**Platform Engineering + FinOps — Enterprise shift to managed K8s** ⚠️

Platform teams สมดุล 3 แรง: (1) delivery speed ต้อง faster, (2) cost governance จาก CFO, (3) AI/GPU complexity โน้มน้าว ไปทาง managed Kubernetes (EKS, GKE, AKS) และ platform-as-a-service layers (Pulumi, Supabase, Modal) สำหรับ compute-as-code ลดการ lock resource ลง ([Medium - Orlando Hurtado](https://medium.com/@orlando1409/beyond-kubernetes-platform-engineering-trends-for-2026-8f82e09e27e0))

→ **ประเด็น:** AI workload เป็น primary workload บน K8s แล้ว ไม่ใช่ optional; FinOps governance เป็นแรง competitive ที่จริง

## 🌐 Web / Frameworks / Languages

**Node.js 22.18.0 native TypeScript support — run .ts files directly ปลายมิ.ย.** [official]

Node.js 22.18.0 ปล่อย stable TypeScript support เมื่อปลายมิ.ย. 2026 ให้ developers รันไฟล์ `.ts` โดยตรงบน Node.js ไม่ต้อง transpiler build step ใช้คำสั่ง `node --experimental-strip-types script.ts` ซึ่งเป็น incremental จาก experimental flag ก่อนหน้า TypeScript integration นี้ลบขั้นตอน dev experience ออก, ลดเวลา feedback loop สำหรับ development ([Node.js Blog](https://nodejs.org/en/blog/release/v22.18.0/) · [TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes.html))

**State of JavaScript 2025 — 40% developers use TypeScript exclusively, rolldown momentum** ⚠️

State of JavaScript 2025 (ตีพิมพ์ มิ.ย. 2026) พบ 40% developers ใช้ TypeScript exclusively เพิ่มจาก 28% ในปี 2024 Rust-based rolldown bundler ได้ momentum จาก early adopters: build time ลดจาก 2.5m เหลือ 40s; Vite ครองด้าน 98% satisfaction; Next.js อันดับ 1 framework สำหรับ production globally ([StateOfJS 2025](https://2025.stateofjs.com/) · [InfoQ](https://www.infoq.com/news/2026/03/state-of-js-survey-2025/))

**Web Dev Trends 2026 — SSR, RSC, performance budgets, AI-assisted linting** ⚠️

LogRocket รวบรวม 8 web dev trends: SSR กลับมาเป็น default (แทน SPA), React Server Components (RSC) ลด JavaScript payload, performance budgets ขึ้นเป็นต้นทาง checklist, TypeScript as baseline, AI-assisted code generation ใน linting (ไม่ logic), monorepos with Turbo/Nx เป็น standard ([LogRocket Blog](https://blog.logrocket.com/8-trends-web-dev-2026/))

→ **พรรณนา:** Node.js native TypeScript ลบขั้นตอนพัฒนา; build tool landscape rolldown vs Vite ยังอยู่ maturity race
